Glossary

Scoring Ratings

OPR — Offensive Power Rating
How many points a team contributes to their alliance per match on average. Solved via weighted least squares across all alliance rows simultaneously to separate each team's individual contribution.
AOPR — Adjusted OPR (headline stat)
OPR with defensive suppression credited back — reveals a team's true offensive ceiling when defense held them down. Computed by proportionally distributing per-match refund amounts back to each team based on their OPR share.
Δ Delta — Defensive Impact
How many points per match defense costs this team on average, measured only over matches where they were actively defended. Averaged exclusively over defended matches (not all matches) to preserve the true magnitude rather than diluting it.
Avg Contribution
A team's estimated share of their alliance's actual score per match. Each alliance score is split in proportion to each robot's OPR, then averaged across the season.
Event OPR
OPR restricted to matches from the currently selected event. Uses the same least-squares solve but built from that event's match matrix only.
Synergy
A measure of how often a team's alliance outperforms what their combined ratings predict. Computed as the average positive residual across all matches the team played in.

Defensive Ratings

DPR — Defensive Power Rating
How many points a team suppresses from the opposing alliance per match. Uses the same least-squares solve as OPR, targeting the points-below-expectation vector for the opposing side.
WDPR — Weighted Defensive Power Rating
DPR where each match row is weighted by the opposing alliance's combined OPR, so suppressing a strong alliance counts for more. Produces higher values for defenders who consistently stop top-ranked opponents.
Event DPR
DPR restricted to the currently selected event. Built from that event's match matrix only, so it reflects single-event defensive performance.
🛡 Defender (role badge)
Assigned when DPR exceeds 1.5× OPR across at least 6 matches. The threshold ensures the classification requires both a meaningful defensive rating and enough sample size to be reliable.

Match-Level Concepts

Residual
The difference between a team's predicted alliance score and the actual score — positive means they underperformed, negative means they outperformed. Used to detect defensive suppression and anomalous matches.
Refund
Points credited back when a defender on the opposing side is linked to a positive residual. Capped at the residual value so it never exceeds actual underperformance.
Contribution Breakdown
Per-match score estimate for one team, shown in the detail panel. Computed as that team's OPR share of the alliance's actual score each match.
Consistency
How stable a team's per-match contributions are, on a 0–1 scale. Computed as 1 / (1 + std dev of contributions) — closer to 1.0 means more predictable output.
⚡ OPR Breaker
A match where the alliance scored far below expectation due to a breakdown, penalty, or anomaly. Row weight is reduced to 0.1 so it doesn't distort the season solve.
Variability
Match-to-match scoring instability for a team. Computed as the standard deviation of residuals across all rows that team appeared in.

How the Math Works

Why not just use average score?
Raw averages can't separate what each robot contributed since three robots share one alliance score. OPR solves a system of equations across all matches at once to isolate each team's individual output.
Match Weighting
Each match row is weighted by recency, playoff stage, and event tier so recent and high-stakes matches influence ratings more. Weights are applied as a diagonal matrix before the least-squares solve.
Ridge λ (shown in stat bar)
A damping term added to the solver when the data matrix is poorly conditioned. Selected adaptively based on the condition number of the weighted design matrix; can be overridden explicitly via config.
Δ Raw vs Δ Avg
Δ Raw (shown in the table) averages the refund only over defended matches; Δ Avg (AOPR − OPR) spreads it across all matches. Raw preserves impact magnitude; avg dilutes it for teams who were only occasionally targeted.
Solver Caching
Results are skipped entirely if the dataset fingerprint and config hash match the previous run. Individual matrix solves are also cached within a run to avoid recomputing identical systems.
Config Parameters
Ridge λ, Noise σ, and Min Matches control solver stability, anomaly exclusion, and ranking eligibility respectively. All three are hashed into the pipeline cache key and stored in solve metadata.
Reproducibility Metadata
Every solve stores a dataset hash, config snapshot, and timestamp. Any result can be re-traced to the exact match data and parameters that produced it.
Low Match Warning
Shown when a team has fewer than 6 matches. Ratings are statistically unreliable below this threshold and should be treated as estimates.
